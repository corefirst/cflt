# 槽位消歧参考

> **版本：** 1.0.0（内部草案）
> **作者：** CFLT 核心团队
> **组织：** [CFLT.center](https://cflt.center)

> **目的：** 一份规范性参考，用于把任意话语中的修饰语唯一地分配到 CFLT 的四个位置之一（或识别为核心内部成分）。本文档是人类课程作者、AI 标注员、以及逻辑 Transformer 引擎的**操作层 ground truth**。

---

## 1. 本文档为何存在

四元规范序列 `[核心] → [理由] → [空间] → [时间]` **只对场景框架完备**（事件外部世界）。事件核本身还包含 —— 方式、工具、受益者、伴随、情态、否定 —— 这些在结构上属于槽位 0（核心），不是独立槽位。

本文档提供：
- 一棵**判定树**用于无歧义地路由任何修饰语（§2）
- 一张**50 例参考表**覆盖最常见的边界案例（§3）
- 模糊组合的**边界规则**（§4）
- 跨语言一致性检验，证明规则同时适用于 SVO / SOV / VSO / 主题优先类型（§5）

两层模型（事件核 vs 场景框架）的理论基础，参见 [`../foundations/core-concept.md`](../foundations/core-concept.md) §2.1–§2.2。

## 2. 判定流程（启发式，带优先级层级）

> **本流程的状态。** 这是一份**带优先级的启发式**，不是经过形式验证的判定树。形式完备性（每个修饰语对每种输入组合都映射到唯一槽位）是开放问题 —— 参见 §8 开放边界。该流程足以应对绝大多数自然语言修饰语；两个问题都回答"是"的边界案例由下方的优先级层级解决。

对任何词或短语，按以下顺序自问：

1. **它是谓词或谓词的核心论元吗？**（主语、直接/间接宾语）→ **事件核内**
2. **它修饰动作的执行方式吗？**（方式、工具、受益者、伴随、情态、否定）→ **事件核内**
   > *作用域说明。* 对情态与否定而言，"怎样"这一问法是 **CFLT 操作启发式**，而非类型学检验。跨语言地看，情态与否定算子的*语义作用域*与*句法附着*差异显著（Horn 1989；Palmer 2001；Miestamo 2005；van der Auwera & Plungian 1998）—— 句内 vs 句末、谓词层 vs 命题层。CFLT 把它们路由进 Core 的约定是一项项目自定的放置策略，需针对各语言的作用域与形态加以检验，而非主张它们普遍"回答怎样"或附着于谓词。
3. **它回答"为什么"吗？**（原因 / 目的 / 条件）→ **槽位 1 [理由]**
4. **它回答"哪里"吗？**（物理 / domain / 媒介）→ **槽位 2 [空间]**
5. **它回答"何时 / 多久一次 / 多长时间"吗？** → **槽位 3 [时间]**
6. **以上都不是** —— 见 §4 边界案例。

**层级优先**：1 > 2 > 3 > 4 > 5。如果某短语合理地适合多个类别，编号小的胜出（即"事件核内"胜过"场景框架"）。

```mermaid
graph TD
    Start[待分类的修饰语] --> Q1{是谓词或核心论元？}
    Q1 -- 是 --> Core[事件核内]
    Q1 -- 否 --> Q2{修饰动作的执行方式？<br/>方式/工具/受益者/伴随/情态/否定}
    Q2 -- 是 --> Core
    Q2 -- 否 --> Q3{回答"为什么"？<br/>原因/目的/条件}
    Q3 -- 是 --> R[槽位 1：理由]
    Q3 -- 否 --> Q4{回答"哪里"？<br/>物理/domain/媒介}
    Q4 -- 是 --> S[槽位 2：空间]
    Q4 -- 否 --> Q5{回答"何时/多久一次/多长时间"？}
    Q5 -- 是 --> T[槽位 3：时间]
    Q5 -- 否 --> Edge[见 §4 边界案例]

    style Core fill:#f96,stroke:#333,stroke-width:2px
```

## 4. 边界案例与消歧规则

### 4.1 "当 X 时" —— 时间还是理由？

*"当你到达的时候"* 可以是：
- **时间** —— *"当你离开时我很难过。"*（= 在那个时刻）—— 时间定位
- **理由（条件）** —— *"等你到了我就走。"*（= 一旦 / 如果）—— 条件依赖

**规则**：如果主句是**未来**或**情态**（"将"、"会"、"能"），那么 *当*-从句是条件 → **理由**。如果主句是**过去陈述**描述同时性，→ **时间**。

### 4.2 "通过 X" —— 工具还是时间？

- *"通过电话"* —— 工具 → **事件核内**
- *"截至周二"* —— 时间截止 → **时间**

**规则**：如果 X 是实体/工具/通道 → 工具（核心）。如果 X 是时间锚 → 时间。

### 4.3 "在 X" —— 空间还是时间？

- *"在厨房"* —— 物理 → **空间**
- *"在 2026 年"* —— 时间 → **时间**
- *"在软件领域"* —— 抽象 domain → **空间**
- *"在三分钟内"* —— 时段 → **时间**

**规则**：如果 X 表示区域/领域/容器 → 空间。如果 X 表示年份/时长/时段 → 时间。

### 4.4 "为 X" —— 受益者、目的、还是时段？

- *"为妈妈"* —— 受益者 → **事件核内**
- *"为了学习"* —— 目的 → **理由**
- *"持续一小时"* —— 时段 → **时间**

**规则**：如果 X 是人/实体 → 受益者（核心）。如果 X 是活动/目标 → 目的（理由）。如果 X 是时段 → 时间。

### 4.5 同槽多元素

如果一个句子在同一槽位有**多个**元素，按规范顺序相邻放置：

- **多个理由元素**：原因后接条件。*"……因为我累了，如果你非要知道的话，……"*
- **多个时间元素**：时频 → 时段 → 时点。*"每天一小时三点钟"*
- **多个空间元素**：从大到小。*"在旧金山，办公室里，桌上"*

### 4.6 话语连接词（在所有槽位之外）

*"然而、因此、然后、所以、反正、顺便说一下"* —— 这些**不**填入任何四槽。它们出现在：
- 块与块之间（参见 [`complex-structures.md`](./complex-structures.md) §3 的块级链接），或
- 核心之前作为话语锚（如 *"顺便说一下，我五点走。"*）

### 4.7 时态不是槽位

谓词的语法时态（过去、现在、将来）**在事件核内**，附着于动词形式。时间槽承载时间状语；时态形态由时间令牌触发，但在事件核内通过词汇手段实现。参见 [`human-learning.md`](./human-learning.md) §2.1 关于后向时间约束的论述。

## 5. 跨语言一致性检验

判定树意在跨语言类型适用。下面是最小验证。

> **缩写提醒。** **S = Subject（主语）、V = Verb（动词）、O = Object（宾语）**。SVO / SOV / VSO 表示及物陈述句中这些成分的默认表面语序。完整类型学说明参见 [`../foundations/linguistics.md`](../foundations/linguistics.md) §1.1。

### 5.1 SVO —— 英语

> *"I baked the cake with butter, slowly, for my mom, in the kitchen, yesterday, because it was her birthday."*

| 成分 | 槽位 |
|---|---|
| I baked the cake | 核心（谓词 + 受事） |
| with butter | 核心（工具） |
| slowly | 核心（方式） |
| for my mom | 核心（受益者） |
| in the kitchen | 空间 |
| yesterday | 时间 |
| because it was her birthday | 理由 |

**CFLT 规范重排**（事件核作为一个语块在前，然后场景框架 R→S→T）：
> *"I baked the cake with butter slowly for my mom, because it was her birthday, in the kitchen, yesterday."*

### 5.2 SOV —— 日语

> *昨日、母のために、台所で、誕生日だったので、バターでゆっくりケーキを焼いた。*

| 成分 | 槽位 |
|---|---|
| バターでゆっくりケーキを焼いた | 核心（工具 *de* + 方式 + 受事 *o* + 动词） |
| 母のために | 核心（受益者，与格 + 目的） |
| 誕生日だったので | 理由 |
| 台所で | 空间 |
| 昨日 | 时间 |

**CFLT 规范重排**（事件核作为一个语块在前，然后 R→S→T）—— 注意动词按日语句法仍居末位；CFLT 只规定状语相对于事件核边界的顺序：
> *母のためにバターでゆっくりケーキを焼いた、誕生日だったので、台所で、昨日。*

> **关于日语标点。** 句末动词形（-た）后用读点续接是**事后追加/右移位**，在日语口语中常见，可作为 CFLT 教学示意接受。正式书面日语通常会把同样的内容分为独立句子（如：*母のためにバターでゆっくりケーキを焼いた。誕生日だったので、台所で焼いた。昨日のことだ。*）。CFLT-日语渲染应匹配语篇语域。

### 5.3 主题优先 —— 中文

> *昨天，因为是她生日，在厨房里，他用黄油慢慢地为妈妈烤了一个蛋糕。*

CFLT 规范重排：
> *他用黄油慢慢地为妈妈烤了一个蛋糕，因为是她生日，在厨房里，昨天。*

事件核 *"用黄油慢慢地为妈妈烤了一个蛋糕"*（工具 + 方式 + 受益者 + 谓词-受事）保持其汉语原生顺序；CFLT 只把 R/S/T 框架重排到事件核**之后**。

### 5.4 VSO —— 威尔士语（示意）

威尔士语规范：V-S-O。CFLT 把 R/S/T 移到 V-S-O 事件核之后，保持该语言的动词起首结构不变。

## 6. 本参考**不**做什么

1. 它**不**替代每种语言的句法。核心内部的组装（格标、助词、介词、核心内部的语序）由 L1/L2 语法支配，不归 CFLT。
2. 它**不**规定有标话语中的风格化语序。CFLT 定义的是*无标默认值*；修辞性前置、分裂句、主题化保持可用（参见 [`../foundations/core-concept.md`](../foundations/core-concept.md) §6）。
3. 它**不**处理引语或嵌入命题态度动词（"我认为……"、"她说……"）。嵌套命题参见 [`./complex-structures.md`](./complex-structures.md)。

## 7. 验证流程

扩展本参考时（例如增加新 domain 或新语言）：

1. 新条目必须通过**替换测试**与**听者问题测试**（定义见 [`../foundations/core-concept.md`](../foundations/core-concept.md) §2.2）。
2. 落入 §4 边界案例的新条目必须附消歧规则。
3. 跨语言条目（§5）必须在目标语言中**结构上**正确，不能仅是英语示例的翻译。

## 8. 开放边界（诚实的局限）

以下问题尚未充分规定，列在此处供后续工作处理：

1. **引语与转述**。*"他说'我要走了'昨天"* —— *"我要走了"* 是嵌套 CFLT 块还是直接引语？
2. **情态堆叠**。*"他可能必须已经走了"* —— 多层情态都附着于核心，但其内部顺序是语言特定的。
3. **东亚语言的话语助词**（汉语 吧、呢、啊；日语 ね、よ、か）。它们附着于整句话语，不属于任何单一槽位。视为**话语级**标记，处于四槽体系之外。
4. **韩日语的敬语与礼貌系统**。这些渗透整个话语的形态层，不是槽位特定的。CFLT 将其作为**部分可分离**而非完全正交的层处理：神经与加工证据（Momo 等 2008；Cui 等 2022）表明敬语加工与社会语用及语法因素相互作用，并非作为可拆卸的外包装运行，故与槽位内容的完全独立性尚未确立。"先 CFLT 结构、后敬语层"的两阶段工作流是 CFLT 的**待检验假设**，而非已验证的程序；礼貌层的部分可分离性说明参见 [`../foundations/sociolinguistics.md`](../foundations/sociolinguistics.md)。
