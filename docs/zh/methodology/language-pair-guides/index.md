> **目的：** 把 CFLT 操作化到具体的 L1↔L2 语言对。协议层（核心居首、R-S-T 顺序）与槽位语义层是普适的（参见 [`../../foundations/core-concept.md`](../../foundations/core-concept.md) §2.3）。**不**普适的是事件核内部组装与语言特定的 edge case —— 这些按语言对在此处文档化。

---

## 为什么需要这个系列

两层模型（`core-concept.md` §2.1）解耦了：
- **普适协议**（槽位顺序、槽位语义）—— 适用于所有语言对
- **语言特定的组装**（事件核如何句法上组建）—— 委托给原生 L1/L2 语法

协议适用于任何语言对，但具体语言对的学习者需要：
- 当*他们的* L1 句法遇到*他们的* L2 句法时出现的 edge case 列表
- 不经过不必要绕道的 CFLT 枢轴例句
- 该语言对常见的陷阱列表

这些指南**不是** `slot-disambiguation.md` 的复制 —— 后者给出普适判定树。这里的指南补充该判定树尚未涵盖的**语言对特定的判例法**。

## 如何使用指南

每份指南的结构：

1. **语言对特定说明** —— 该对在类型学上有何独特之处
2. **事件核组装差异** —— 同一 Core 在两种语言中如何不同地构建
3. **edge case** —— 此语言对中与普适默认不同的边界判定
4. **完整例句** —— 5–10 个完全分解的例句，含两种语言的表面形式
5. **常见学习者陷阱** —— L1 说话者瞄准 L2 时通常误用 CFLT 之处

阅读*你具体* L1↔L2 对的指南。普适协议与槽位语义在 foundations 文档中；本指南只补充你所在语言对特有的内容。

## 覆盖路线图

当前四份指南覆盖了全球 L2 市场流量最大的对，以及为捍卫 CFLT 普适性主张所必需的类型学示意对。未来指南可能包括：

- 韩语 ↔ 日语（亚洲内部，皆 SOV 但黏着策略不同）
- 西班牙语 ↔ 英语（罗曼 vs 日耳曼，皆 SVO）
- 阿拉伯语 ↔ 英语（VSO ↔ SVO，词根模式形态学）
- 法语 ↔ 汉语（核心在前的罗曼 vs 核心在前的孤立）

欢迎贡献 —— 每份指南需满足的标准参见[验证流程](../slot-disambiguation.md#7-验证流程)。
