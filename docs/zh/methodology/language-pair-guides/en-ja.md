# 英语 ↔ 日语：CFLT 操作化

> **版本：** 1.0.0（内部草案）
> **许可：** [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)

> **目标读者。** 英语母语的日语学习者，以及日语母语的英语学习者。经典 SVO ↔ SOV 类型对照。

---

> **另见**：[`zh-ja.md`](./zh-ja.md) 提供无英语桥接的汉↔日视角，[`zh-en.md`](./zh-en.md) 提供中文 L1 → 英语 L2 视角。

## 1. 语言对特定的类型学说明

| 属性 | 英语 | 日语 |
|---|---|---|
| **默认词序** | SVO | SOV |
| **中心语方向** | 核心在前（V 在 O 之前、P 在 NP 之前） | 全部核心在后（O 在 V 之前、NP 在 P 之前、修饰在名词之前） |
| **格标？** | 极少（仅在代词上） | 丰富（が、を、に、で、と、から、へ） |
| **时态形态？** | 是 | 是 |
| **主题优先？** | 弱 | 强（带显式 は 标记） |
| **省略主语？** | 否 | 是 |
| **敬语系统？** | 无 | 渗透性（敬語：尊敬、谦逊、礼貌形式） |

**对 CFLT 的关键含义。** 该对最大限度地考验两层模型：协议层对两种语言相同，但事件核内部组装是**反向**的（SVO vs SOV）。没有 CFLT 的两层分离，学习者会混淆这两层，要么在日语句子中保留英语 SVO 顺序（不合语法），要么在动词到达前难以组织话语。

CFLT 的贡献：把"Core 之后是什么"（普适 R-S-T 顺序）与"Core 内部如何结构"（带格标的日语 SOV）分开，使该对以纯模仿不可能的方式变得可学。

## 2. 事件核组装：最大分歧

同一个 Core "我昨天在书店买了一本书" 组装为：

- **英语**：*I bought a book* —— S V O（SVO，核心在前）
- **日语**：*私は本を買った* —— S は O を V（SOV，核心在后，格标）

这些是**最大类型学分歧的 Core** —— 每个结构属性都镜像。CFLT 不对齐它们；每种语言原生地组装自己的 Core。

**CFLT 所做的**：把整个 Core 块放在槽位 0，然后应用普适 R-S-T 框架。

## 3. 语言对特定的 edge case

### 3.1 动词后置纪律（英语 L1 → 日语 L2）

最难的关卡：英语母语者必须在说出所有修饰语时把动词保留在工作记忆中，因为日语动词最后才出现。

**CFLT 缓解**：尽管日语 Core 以动词结尾，Core 块本身仍占据槽位 0。英语母语者先用英语动词中置直觉在脑中规划 Core，然后用日语 SOV 顺序口语化。R-S-T 场景框架不论如何按规范顺序跟随。

**具体训练**：
1. 心理选择 Core 动作（如 "buy the book"）
2. 构建日语 Core：本を買う（book + を + buy）
3. 附加礼貌/体貌：本を買いました
4. 应用场景框架：本を買いました、本屋で、昨日（Core、空间、时间）

### 3.2 は（主题）vs が（主语）

> **教学简化声明。** Kuno (1973) 区分は的*主题型*与*对比型*两用，が的*中性描述*与*穷尽列举*两用 —— 共四种功能，不是两种。Heycock (2008) 与 Kuroda (2005) 进一步记录了边界案例（嵌入子句不能托管主题-は；对比-は 带下降语调等）。下面的五条规则覆盖**主题-は ↔ 穷尽-が** 这一无标默认的主映射，足以应对多数 CFLT 教学场景；它不处理全部四种用法。深入掌握请参见 Kuno 1973（第 2 章）、Heycock 2008（牛津日语语言学手册）、Kuroda 2005（J. East Asian Linguistics 14:1–58）。

日语区分主题标记主语（は）与焦点标记主语（が）。英语无形态标记；它通过词序、对比重音或词汇选择（*as for X*）表面化。

**en→ja 规则**：
- 默认英语主语 → 日语 は（建立主题时）或 が（引入新信息时）
- 英语主语对比重音 → 日语 は 配下降语调，或 が 强调
- "As for X..." → 主题前置 X-は

**ja→en 规则**：
- は → 英语 SVO 中默认主语；如对比，可加 "as for"
- 非强调语境中的 が → 默认英语主语
- 焦点语境中的 が（如 私が書いた "It is I who wrote it"）→ 英语分裂句

### 3.3 日语体貌：-ている、-てある、-ておく

日语有丰富的动词周边形式编码体貌/结果/准备。这些全留在 **Core 内**（动词形态），不在任何场景框架槽。

| 日语 | 功能 | 英语对应 |
|---|---|---|
| -ている | 进行或状态结果 | *is V-ing* 或 *has V-ed*（取决于动词类） |
| -てある | 来自有意动作的结果 | *has been V-ed* |
| -ておく | 准备性动作 | *V in advance* |
| -てしまう | 完成 / 遗憾 | *finish V-ing* / *unfortunately V* |

**规则**：在两种语言中保持这些 Core 内；不要让它们泄漏到时间槽。

### 3.4 敬语（敬語）—— 与 CFLT 部分可分离

日语敬语形态学修饰整个话语。英语无直接对应；礼貌通过词汇（*could you、would you mind*）或情态实现。

**规则**：敬语与四槽协议**部分可分离** —— 它不改变 Core 或场景框架的*分配*，但神经成像显示日语敬语处理动用左 IFG（Momo, Sakai & Sakai 2008，*Brain and Language* 107(1)，81–89），在社会语用敬语一致语境下还额外动用双侧岛叶与背内侧前额叶皮层（地位较低听话者条件，Cui, Jeong, Okamoto, Takahashi, Kawashima & Sugiura 2022，*Journal of Neurolinguistics* 62，101041）。CFLT 教学建议 —— 先做槽位工作、再做敬语层 —— 因此作为*认知工作流*有效，但不应过度声称为完全神经独立；社会认知网络的招募对听话者地位与敬语形式都敏感。详细讨论参见 [`../../foundations/sociolinguistics.md`](../../foundations/sociolinguistics.md)。

对学习者：把 CFLT 槽位工作和敬语工作作为分开的认知任务。先 CFLT 结构，后敬语层。

### 3.5 助词 で、に、と —— 多个槽位候选

单个日语助词可映射到多个 CFLT 槽位：

| 助词 | 可能 CFLT 位置 | 消歧 |
|---|---|---|
| で（工具） | Core 内 | *ペンで書く* —— 工具 |
| で（动作地点） | 槽位 2 [空间] | *公園で会う* —— 地点 |
| に（接受者/与格） | Core 内（受益者） | *母にあげる* —— 接受者 |
| に（存在地点） | 槽位 2 [空间] | *京都に住む* —— 地点 |
| に（时点） | 槽位 3 [时间] | *3時に来る* —— 时间 |
| と（伴随） | Core 内 | *友達と行く* —— 伴随 |
| と（引语标记） | 不是槽位 —— Core 内 | *「来る」と言った* |

**规则**：应用普适替换测试（`slot-disambiguation.md` §2）。助词不决定槽位；回答的**功能性问题**决定。

### 3.6 没有主语？追踪谁缺失

日语自由省略主语；英语不省。CFLT-英语 Core 需要显式主语；CFLT-日语 Core 可省略它们。

**en→ja 规则**：渲染 CFLT-日语时，省略英语主语代词是安全的，只要语境能恢复。

**ja→en 规则**：英语必须重建主语。第一人称日记式语境默认用 *I*；否则用话语语境。

## 4. 完整例句

### 例 1 —— 基本动作

- **英语 CFLT**：*I didn't go out, because it rained, at home, yesterday.*
- **日语 CFLT**：*出かけなかった、雨が降ったから、家で、昨日。*

注：日语省略 "我"（pro-drop）；英语不省。

### 例 2 —— 请求带礼貌

- **英语 CFLT**：*Could you please pass the salt, on the table, now?*
- **日语 CFLT（礼貌）**：*塩を渡してくれませんか、テーブルの上で、今。*
- **日语 CFLT（谦逊礼貌）**：*塩を渡していただけませんか、テーブルの上で、今。*

英语礼貌层（*could you ... please*）映射到多个日语敬语选项；CFLT 不为它们分配槽位。

### 例 3 —— 工具在 Core 内

- **英语 CFLT**：*I cut the meat with a knife, in the kitchen, just now.*
- **日语 CFLT**：*ナイフで肉を切った、台所で、さっき。*

工具 *with a knife* / *ナイフで* 在两种语言中都留在 Core 内，尽管句法激烈不同。

### 例 4 —— 全价位 Core

- **英语 CFLT**：*I baked the cake with butter, slowly, for my mom, in the kitchen, yesterday.*
- **日语 CFLT**：*母のためにバターでゆっくりケーキを焼いた、台所で、昨日。*

注：日语排列 受益者-に → 工具-で → 方式 → 宾语-を → 动词（全部 SOV 内部）；英语排列 S-V-O + 工具 + 方式 + 受益者（全部 SVO 内部）。两者都占据 Core 槽位 0；两者都跟随 空间 + 时间。这里的日语例句省略了理由（"因为是她的生日"）；含 4 槽完整版参见 [`slot-disambiguation.md`](../slot-disambiguation.md) §5.2。

### 例 5 —— 身份 + 描述

- **英语 CFLT**：*That girl is my sister, wearing a red dress, in the photo, from last summer.*
- **日语 CFLT**：*あの女の子は私の妹だ、赤い服を着て、写真の中で、去年の夏。*

は 标记的主语 *あの女の子* 镜像英语 *that girl* 主题。

## 5. 常见学习者陷阱

### 5.1 英语母语者学日语

- **忘记动词后置**：产出 *I bought yesterday a book* SOV-碎片化的日语
- **误用 は vs が**：对语法主语应用 は 而无主题语境
- **过度显式主语**：在语境允许时不省代词
- **敬语级别不匹配**：CFLT 应用正确但社会语境的敬语级别错误

### 5.2 日语母语者学英语

- **SOV 渗入英语**：通过字面槽位翻译产出 *I yesterday book bought*
- **缺失时态**：类似中文 L1 → 英语（用后向时间约束）
- **缺失冠词**：无日语对应
- **直接 は → "is"**：把主题标记与系动词混淆
- **缺失主语**：让英语 Core 无主语

## 6. 验证清单

1. ☐ Core 在槽位 0；场景框架按 R-S-T 顺序
2. ☐ 英语 Core：SVO + 时态；日语 Core：SOV + 格标 + 动词后置
3. ☐ 体貌/时态形态留在 Core 内（不在时间槽）
4. ☐ は/が 用法匹配主题 vs 焦点区分（en→ja 时）
5. ☐ 英语重建主语；日语允许省略
6. ☐ 敬语作为部分可分离层应用；不影响槽位分配
7. ☐ で / に / と 助词到槽位映射通过替换测试验证，不通过助词身份

## 7. 该对的开放问题

1. **日语句末助词**（ね、よ、か、わ、ぞ）—— 话语级，不属于任何槽位
2. **敬语级联** —— 全程使用敬语时，每个 CFLT 块中的每个动词都需标记；这是与 CFLT 正交的重认知负荷
3. **日语引语**用 と 作引语标记 —— 引用内容本身可能形成嵌套 CFLT 块；参见 [`../complex-structures.md`](../complex-structures.md)
