# 汉语 ↔ 日语：无英语桥接的 CFLT

> **版本：** 1.0.0（内部草案）
> **许可：** [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/)

> **本指南为何重要。** 部分中文母语日语学习者（反向亦然）借道英语：中文 → 英语 → 日语，这是 2-跳翻译。CFLT *假设*这种绕道会增加摩擦层与错误累积，而**直接**通过协议层枢轴（无需英语）能减少它们 —— 但英语绕道的普遍程度及其相对成本尚无证据支撑；请将此视为动机，而非已测得的结果。本指南确实证明的是可行性：CFLT 能在一个非印欧语对上运作而无需英语中介。*完整的普适性论证* —— 五种语言、四个语系并附参考语法引用 —— 见 [`../../foundations/core-concept.md`](../../foundations/core-concept.md) §2.5。zh↔ja 对实际证明的是较窄的英语无关性（可行性）主张；英语无关带来的*学习优势*仍是有待检验的假设。

---

> **另见**：[`en-ja.md`](./en-ja.md) 提供日语侧的英语桥接视角，[`en-zh.md`](./en-zh.md) 提供汉语侧的英语桥接视角。

## 1. 语言对特定的类型学说明

| 属性 | 汉语（普通话） | 日语 |
|---|---|---|
| **默认词序** | SVO（主-动-宾） | SOV（主-宾-动） |
| **中心语方向** | 多数核心在前（V-O、P-NP），但 NP 修饰核心在后 | 全部核心在后（O-V、NP-P、修饰-名词） |
| **主题优先？** | 是（强烈） | 是（带显式主题标记 は） |
| **格标？** | 无（依靠词序+连动结构） | 丰富（が、を、に、で、と、から、へ） |
| **时态形态？** | 无（依靠体助词 了、过、着 + 时间副词） | 丰富（-た 过去、-る 非过去、-ている 进行） |
| **省略主语？** | 是（主语常省略） | 是（主语常省略，宾语有时也省略） |

**对 CFLT 的关键含义。** 两种语言共享*主题优先*（重大便利重叠），但在*事件核组装*上分歧严重（动词后置 vs 动词中置、有格标 vs 无格标）。CFLT 协议层（核心居首、R-S-T）对两者完全相同；事件核用各语言自己的机制构建；当某个槽位填充内容难以直接映射时，CFLT 建议借助受 NSM（Wierzbicka 1996）启发的受控语义元语言对其意义进行释义 —— 是 NSM 式的释义，而非逐 token 翻译（NSM 原语是意义，不是可直接替换的 token）。

## 2. 事件核组装差异

同一个 Core "我用钱为妈妈买了一本书" 不同地组装：

### 汉语组装
- 词序：主语-连动词-工具-连动词受益-受益者-动词-宾语
- *我用钱为妈妈买了一本书*
  - 我（S）— 用（连动：with）— 钱（工具）— 为（连动：for）— 妈妈（受益者）— 买（V）— 了（完成体）— 一本书（O）

### 日语组装
- 词序：主语-工具-で-受益者-に-宾语-を-动词
- *私はお金で母に本を買った*
  - 私（S）— は（主题）— お金（工具）— で（格：工具）— 母（受益者）— に（格：与格）— 本（O）— を（格：宾格）— 買った（V + 过去）

**CFLT 规定的**：两个事件核都作为单一显著性单元占据槽位 0。内部排列（汉语连动结构、日语后置格标）**不归 CFLT 管** —— 各语言硬件自行处理。

**CFLT 确实规定的**：事件核闭合后，场景框架槽位（理由、空间、时间）按规范顺序跟随，使用各语言原生从属词/助词附着。

## 3. 语言对特定的 edge case

这些是在 zh-ja 对中与 `slot-disambiguation.md` 的项目标准默认不同的边界判定。

> **范围说明。** 把字句、は/が、主题、对比、语法主语与焦点是**依语境而定的对应，不是直接等价物**。尤其 は/が 的选择涉及主题、对比、焦点、穷尽性、韵律与子句类型之间的复杂交互（Kuno 1973；Kuroda 2005；Heycock 2008），因此一个把字句受事、一个主题或一个被焦点化的主语，并不会按规则映射到单一的日语或汉语形式。请把下面的箭头读作有待结合语境与母语者判断确认的默认起始假设，而非确定性映射。

### 3.1 汉语 把 (bǎ) 字句 → 日语主题化

汉语 **把** 字句把定指宾语前置到动词前：*我把书买了*。表面词序：S-把-O-V。功能上，这把*受事*相对于动作前景化。这在 CFLT 术语中**位于 Core 内部** —— 属于汉语事件核构建方式的一部分。

功能对应的日语常用主题标记：*本は私が買った*（字面 "对于这本书，我买了"）。同样的 Core 内前景化，由 は 标记实现。

**zh→ja 默认**（依语境而定，须结合语境确认）：如果汉语把字句受事是话题/已知指称，は 往往是日语中较自然的渲染；如果是对比性焦点，が 或默认 を 标记宾语可能更合适。这些是倾向，不是一对一的规则。

### 3.2 日语 は（主题）vs が（主语）→ 汉语主题化 vs SVO

日语系统性地区分主题（は）和语法主语（が）。汉语没有这一区别的形态标记 —— 它依靠词序（被主题化的指称移到子句首位）。

- 日：*この本は私が読んだ*（对于这本书，我读了）
- 中：*这本书我读了*（字面主题前置）
- 中：*我读了这本书*（默认 SVO；主题性较弱）

**ja→zh 默认**（依语境而定，须结合语境确认）：は 标记的主语往往渲染为汉语主题前置形式（通常在 Core 之前）；が 标记的主语往往渲染为默认 SVO 位置。这些是倾向，不是等价 —— 选择仍取决于语境中的主题/对比/焦点。**警告**：不要盲目把 は 译为 是（系动词）—— 它们无关。

### 3.3 汉语 因为/所以 vs 日语 から/ので/ため（理由槽位）

汉语通常用 因为 X、所以 Y 或后置 因为 X 显式标记理由。日语有多个理由助词（から 因果、ので 解释、ため 目的）。CFLT [理由] 槽位容纳所有这些，但日语助词的选择带语用权重。

- 中：*我没出去，因为下雨，在家，昨天*（CFLT 形式）
- 日：*出かけなかった、雨が降ったから、家で、昨日*（CFLT 形式）

**规则**：在日语 CFLT 中渲染汉语 因为 从句时，原始因果推理选 から，听者应接受为给定原因时选 ので，目的性时选 ため。

### 3.4 时间与体貌：了/过/着 ↔ -た/-ている/-てある

汉语用**体貌助词**（无时态形态）。日语用**时态+体貌形态**附着于动词。两者都属于 **Core 内部**（附着于谓词），不属于时间槽。时间槽只承载状语性时间引用。

- 中：*昨天* + V + *了*（昨天 + 动词 + 完成体）
- 日：V + *-た* + *昨日*（动词-过去 + 昨天）

**规则**：体貌/时态标记留在 Core 内部（附于动词）；指示性时间词进入时间槽。

### 3.5 没有主语？没问题 —— 但要追踪谁缺失

两种语言都自由省略主语。CFLT 不惩罚省略：如果主语可从语境恢复，Core 可以从谓词开始。

- 中：*买了书*（buy-PERF book）
- 日：*本を買った*（book-ACC bought）
- 两者依然满足"Core 是显著性锚点" —— 谓词承载显著性，即便没有显式主语。

## 4. 完整例句（不经英语）

### 例 1 —— 简单动作带原因

**汉语（默认会话式）：** *昨天下雨，我没出去。*

**第 1 步 —— 识别 Core（替换测试）**：我没出去 是显著事件。下雨 是原因。

**第 2 步 —— CFLT 分解（语言中性）**：
- Core：我没出去 / 出かけなかった
- 理由：因为下雨 / 雨が降ったから
- 空间：在家 / 家で
- 时间：昨天 / 昨日

**第 3 步 —— 渲染汉语 CFLT**：*我没出去，因为下雨，在家，昨天。*

**第 4 步 —— 渲染日语 CFLT**：*出かけなかった、雨が降ったから、家で、昨日。*

每一步都没有英语。

### 例 2 —— 请求带礼貌层

**汉语：** *现在能在桌上帮我递一下盐吗？*

**CFLT 分解**：
- Core（指令）：帮我递一下盐 / 塩を渡してくれ
- 礼貌：能…吗 / -てくれませんか
- 空间：在桌上 / テーブルの上で
- 时间：现在 / 今

**汉语 CFLT**：*请帮我递一下盐，在桌上，现在。*

**日语 CFLT**：*塩を渡してくれませんか、テーブルの上で、今。*

礼貌标记（汉语 能…吗、日语 -てくれませんか）附着于 Core 内部（指令型）。它们不是单独的槽位。

### 例 3 —— 身份陈述带描述

**汉语：** *那个穿红衣服的女孩是我妹妹。*

**CFLT 分解**：
- Core（系属）：那个女孩是我妹妹 / あの女の子は私の妹だ
- 描述（Core 内，类似方式状语）：穿着红衣服 / 赤い服を着ている
- 空间：在照片里 / 写真の中で
- 时间：去年夏天 / 去年の夏

**汉语 CFLT**：*那个女孩是我妹妹，穿着红衣服，在照片里，去年夏天。*

**日语 CFLT**：*あの女の子は私の妹だ、赤い服を着て、写真の中で、去年の夏。*

两种语言中"穿着红衣服"都作为身份主张的分词修饰留在 Core 内部。

### 例 4 —— 工具+方式+受益者全部在 Core 内

**汉语：** *他用筷子在厨房里慢慢地为妈妈做了一道菜，昨天，因为是她生日。*

**CFLT 分解**：
- Core（事件，完整组装）：他用筷子慢慢地为妈妈做了一道菜 / 彼は箸でゆっくり母のために料理を作った
- 理由：因为是她生日 / 母の誕生日だったから
- 空间：在厨房里 / 台所で
- 时间：昨天 / 昨日

**汉语 CFLT**：*他用筷子慢慢地为妈妈做了一道菜，因为是她生日，在厨房里，昨天。*

**日语 CFLT**：*彼は箸でゆっくり母のために料理を作った、母の誕生日だったから、台所で、昨日。*

四个"Core 内"元素（工具 用筷子/箸で、方式 慢慢/ゆっくり、受益者 为妈妈/母のために、动作+体貌 做了/作った）全在 Core 内。三个场景框架槽按规范顺序跟随。

### 例 5 —— 条件型 Core（CFLT-Complex）

**汉语：** *如果你完成报告，我明天就在办公室请你喝咖啡。*

**CFLT 分解**：
- Core（事件）：我请你喝咖啡 / コーヒーをおごる
- 理由（条件）：如果你完成报告 / 報告書を仕上げたら
- 空间：在办公室 / オフィスで
- 时间：明天 / 明日

**汉语 CFLT**：*我请你喝咖啡，如果你完成报告，在办公室，明天。*

**日语 CFLT**：*コーヒーをおごる、報告書を仕上げたら、オフィスで、明日。*

条件（功能 WHY 类）在两种语言中都位于 [理由] 槽位，分别由 如果 / たら / ば 标记。

## 5. 常见学习者陷阱

### 5.1 中文母语者学日语：忘记动词后置纪律

中文母语者产出 CFLT-日语时有时把汉语 SVO 顺序保留，只是在每个汉语词后追加 は/が/を。结果：不合语法的日语。

**诊断**：CFLT 协议被满足（Core 槽位 0），但 Core 的**内部组装**错了 —— 日语 Core 内部要求动词后置。CFLT 不取代该要求。

**修正**：应用场景框架前，先检查 Core 的内部动词是否在 Core 块的末尾。CFLT 只规定 Core 与场景框架之间的边界，不规定内部顺序。

### 5.2 日语母语者学汉语：在汉语省略时仍保持 は/が 区分

日语母语者产出 CFLT-汉语时，有时尝试通过插入 是（"是"）或过度使用主题前置来保留 は/が 区别。结果：中文听起来生硬。

**修正**：当日语句子有 は（主题）时，在汉语中前置主题（移到 Core 之前）。当有 が（主语焦点或新信息）时，保持默认 SVO 顺序。决不用 是 标记主题。

### 5.3 双向：当源语言合并时强行选定一个场景框架槽位

日语 で 可标工具或地点（取决于语境）；汉语 在 标地点而 用 标工具。一个日语 で 可对应汉语 在（空间）或 用（Core 内，工具）。机械翻译会丢失槽位区别。

**诊断**：这正是普适判定树正确处理的 edge case（替换测试：改变 で-NP —— 事件改变吗？是 → 工具，Core 内；否 → 地点，空间槽）。

**修正**：即便源和目标都不是英语，也运行 `slot-disambiguation.md` §2 的判定树。该树是语言无关的。

### 5.4 体貌标记泄漏到时间槽

常见错误：把完成体 了 / -た 放进时间槽。它们不是状语性时间引用 —— 它们是动词形态，附着于 Core 内的谓词。

**规则**：了、过、着、-た、-ている、-ていた 全留在 Core 内。时间槽只承载 昨天 / 今 / 来年 类状语。

## 6. 验证清单

对任何 zh→ja 或 ja→zh 的 CFLT 渲染，验证：

1. ☐ 槽位 0 的 Core 包含显著谓词 + 价位 + 方式（替换测试通过）
2. ☐ Core 内部顺序遵守目标语言句法（日语动词后置、汉语动词中置）
3. ☐ 助词/连动词/格标在 Core 内正确应用
4. ☐ 场景框架按规范顺序跟随：理由 → 空间 → 时间
5. ☐ 体貌/时态标记在 Core 内，不在时间槽
6. ☐ 主题结构（は / 主题前置）跨语言保留
7. ☐ 礼貌层完整（特别是日语，省略可能不礼貌）

7 项全过 —— CFLT 渲染在该语言对中正确，且重要的是，**每一步都没有借道英语**。

## 7. 该对的开放问题

1. **日语敬语形态**繁重且渗透整个话语。CFLT 不为其分配槽位；这是与四槽正交的层（参见 [`../../foundations/sociolinguistics.md`](../../foundations/sociolinguistics.md)）。
2. **话语助词**（汉语 吧/呢/啊、日语 ね/よ/か）是话语级，不属于任何槽位。两种语言都重度使用；CFLT 渲染输出应在话语边界附加它们，不在任何槽位内。
3. **量词 / 计数器**（汉语 一**本**书、日语 本一**冊**）是 NP 内部句法，完全委托给语言原生机制；CFLT 不处理它们。
