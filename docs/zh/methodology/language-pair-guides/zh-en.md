## 1. 语言对特定说明（中文 L1 视角）

类型学对照表参见 [`en-zh.md`](./en-zh.md) §1。

中文母语英语学习者需要内化的关键事实：

1. **英语有时态；汉语没有。** 每个英语有限动词带过去/现在/将来。CFLT-英语 Core *必须*带时态标记。
2. **英语要求显式主语。** 汉语自由省略主语；英语不行。CFLT-英语 Core 需要显式主语。
3. **英语有冠词（a/the）；汉语没有。** 冠词选择是 Core 内的英语问题 —— CFLT 不为其分配槽位但也不能忽视。
4. **英语有复数标记（-s）；汉语只对有生名词用 们。** 与冠词相同：Core 内英语 NP 细节。
5. **英语不激进地主题前置。** 许多自然汉语句子（那本书我看了 / 这件事我处理）渲染到英语时需要重组为 SVO。

## 2. "后向时间约束" —— CFLT 的时态解决方案

这是该语言对最重要的 CFLT 教学机制。完整论述参见 [`../human-learning.md`](../human-learning.md) §2.1。

**问题**：中文 L1 说话者产出 CFLT-英语时常说 *I go to school yesterday* —— CFLT 顺序正确，英语时态错误。他们忘了变形动词。

**CFLT 解决方案**：训练学习者把时间槽中的 *yesterday* 视为**追溯触发器**，反向触发 Core 动词形式（*go → went*）。最初这在句子说出后有意识地发生；练习后，大脑会前向预回响该触发器，在产生时即变形动词。

**操作规则**：产出 CFLT-英语句子后，扫描时间槽。如果含过去指示词（*yesterday、last week、just now*），检查 Core 动词是否过去时变形。如果含未来指示词（*tomorrow、next year*），检查 Core 是否带 *will* 或未来周边形式。

CFLT *预测*这把**选择问题**（中文 L1 没有时态可借鉴）转化为**簿记规则**（检查时间槽，反向验证动词）—— 这是有待检验的设计假设，不是已证实的结果。

> **诚实范围（Lardiere 警示）。** Lardiere (1998) "Patty" 案例研究（DOI 10.1191/026765898674105303）显示，即使一位高级中文 L1 学习者，英语过去时形态产出仅约 6% —— 瓶颈在*产出*，不是*选择*。所述原因为：(a) 汉语音节结构约束（无尾辅音簇）抑制 *-ed* 表层形式；(b) Hawkins & Liszka 2003 *韵律迁移假说* 与 Goad & White 2006 记录的"缺失表层屈折"挑战。CFLT *预测*后向时间约束能应对*选哪个时态*的选择问题（这是设计假设，尚未经干预研究验证）；即便如此，它也**不**应对*如何产出屈折*的问题。有效的汉→英时态教学必须把 CFLT 时间支架与显式语音及形态训练结合。

## 3. 语言对特定的 edge case（中文 L1 → 英语 L2）

### 3.1 是 ("be") 陷阱

汉语 是 = 英语系动词 *be*，但只在身份陈述中：她**是**老师 ↔ She **is** a teacher。形容词谓语中，汉语省略 是：她**很**漂亮 ↔ She **is** beautiful（英语保留 *be*）。

**中文 L1 陷阱**：在英语形容词谓语中省略 *be*（*She beautiful*）。

**修正**：CFLT-英语中，形容词为首的状态 Core 总是带 *be*。*I'm exhausted* 而不是 *I exhausted*。

### 3.2 体貌 ↔ 时态不匹配

汉语 了 ≠ 英语过去时。了 标记事件完成；过去时标记事件时间的前置性。三个粗略映射：

| 汉语 | 英语 | 原因 |
|---|---|---|
| 我吃了（eat-PERF） | I have eaten / I ate | 完成体；英语用现在完成或简单过去取决于语境 |
| 我吃过（eat-EXP） | I have eaten (before) | 经验性 —— 必须用现在完成 |
| 我在吃（PROG eat） | I am eating | 进行体 —— 两种语言都有 |

**规则**：在 CFLT-英语中渲染汉语 了 时，如果时间状语过去指示性（*yesterday*）选简单过去，如果时间不指定或经验性选现在完成。

### 3.3 汉语量词 → 英语裸名词或带冠词的 NP

汉语要求量词：一**本**书（one [量词] book）。英语要求冠词或不带限定词：*a book* / *books*。无干净对齐。

**规则**：CFLT 不为量词/冠词分配槽位 —— 这些是 Core 内的 NP 细节，由各语言语法处理。需要意识，但不是 CFLT 协议问题。

### 3.4 主题前置的汉语 → SVO 英语

中文学习者常尝试在英语中保留主题前置：*That book, I read it three times*。这合语法但在英语中带标（很非正式或对比）。默认英语：*I read that book three times*。

**规则**：在 CFLT-英语中渲染 CFLT-汉语时，默认 SVO。仅当原汉语带强对比焦点时使用英语主题前置。

### 3.5 关系子句方向反转

汉语关系子句先于名词：我看的那本书（I read [REL] that book）。英语关系子句跟随：*the book that I read*。这是 Core 内 NP 问题（CFLT 不控制 NP 顺序），但值得标记，因为中文 L1 说话者常产出 *I read book* + 关系子句位置错误。

**规则**：这是纯目标语言 NP 语法；CFLT 完全委托。Core 槽位被满足，只要显著性锚点（动作或身份）占据位置 0；Core 内 NP 如何构造是目标语言事务。

## 4. 完整例句（zh L1 → en L2）

### 例 1

- **L1（原始）**：*昨天我没出去，因为下雨。*
- **CFLT-汉语**：*我没出去，因为下雨，在家，昨天。*
- **CFLT-英语**：*I didn't go out, because it rained, at home, yesterday.*
- ✅ 时态检查：*yesterday* 触发 *did + not + go out* 和 *rained*（都过去）。

### 例 2

- **L1（原始）**：*我每天早上在公园跑步。*
- **CFLT-汉语**：*我跑步，在公园，每天早上。*
- **CFLT-英语**：*I run, in the park, every morning.*
- 注：现在习惯性；无过去触发器，所以 *run*（不是 *ran*）。

### 例 3 —— 常见错误模式

- **L1（原始）**：*昨天他给我打电话。*
- **错误 CFLT-英语（中文 L1 典型）**：*He call me, yesterday.*（缺时态）
- **正确 CFLT-英语**：*He called me, yesterday.*

时间槽 *yesterday* 必须反向验证 *call → called*。

### 例 4 —— 系动词陷阱

- **L1（原始）**：*我累了，因为开会。*
- **CFLT-汉语**：*我累了，因为开会，在办公室，刚才。*
- **错误 CFLT-英语**：*I exhausted, because of the meeting...*
- **正确 CFLT-英语**：*I'm exhausted, because of the meeting, in the office, just now.*

状态型 Core 要求 *be*。

### 例 5 —— 完成体

- **L1（原始）**：*我看过这本书。*
- **CFLT-汉语**：*我看过这本书。*
- **CFLT-英语**：*I have read this book.*

汉语 过 → 英语现在完成。

## 5. 常见陷阱回顾

| 陷阱 | 频率 | 修正 |
|---|---|---|
| 英语动词缺时态 | 极高 | 后向时间约束 —— 检查时间槽 |
| 状态型 Core 缺系动词 *be* | 高 | 总是为形容词/身份谓语插入 *be* |
| 错误冠词（a vs the vs ∅） | 高 | Core 内 NP 语法；与槽位无关 |
| 在英语中过度使用主题前置 | 中 | 默认 SVO；仅强对比时主题化 |
| 了 → 过去时（而非完成） | 中 | 基于时间状语指示性选时态 |
| 复数 -s 省略 | 中 | Core 内 NP 语法；追踪可数名词 |

## 6. 验证清单

1. ☐ Core 在槽位 0；场景框架按 R-S-T 顺序
2. ☐ 英语 Core 有显式主语（不省略）
3. ☐ 动词时态匹配时间槽指示性（后向时间约束）
4. ☐ 状态型 / 身份型 Core 带系动词 *be*
5. ☐ 体助词 了/过/着 **不**译为时间槽副词 —— 它们触发英语动词的时态/体貌
6. ☐ 冠词与复数标记在 Core 内 NP 层级检查
